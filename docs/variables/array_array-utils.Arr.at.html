    Safely accesses an array element at the specified index, supporting negative indices.

    This function provides type-safe array element access with Optional return type, eliminating the risk of undefined access errors. It supports both positive and negative indices, where negative indices count from the end of the array.

    Index Behavior:

    • Positive indices: 0 is the first element, 1 is the second, etc.
    • Negative indices: -1 is the last element, -2 is the second-to-last, etc.
    • Out-of-bounds indices (either direction): returns Optional.None

    Curried Usage: This function supports currying - when called with only an index, it returns a function that can be applied to arrays, making it ideal for functional composition.

      • Safely retrieves an element at a given index from an array, returning an Optional.

        This function provides type-safe array access with support for negative indexing (e.g., -1 for the last element). Unlike direct array access which can return undefined for out-of-bounds indices, this function always returns a well-typed Optional that explicitly represents the possibility of absence.

        Negative Indexing: Negative indices count from the end of the array:

        • -1 refers to the last element
        • -2 refers to the second-to-last element, etc.

        Curried Usage: This function supports currying - when called with only an index, it returns a function that can be applied to arrays, making it ideal for use in pipe operations.

        Optional Return Type: The return type is always Optional which provides:

        • Type-safe access without undefined in your business logic
        • Explicit handling of "not found" cases
        • Composable error handling with Optional utilities

        Type Parameters

        • const Ar extends readonly unknown[]

        Parameters

        • array: Ar

          The array to access (when using direct call syntax).

        • index: ArgArrayIndexWithNegative<Ar>

          The index to access. Must be a branded SizeType.ArgArr (safe integer). Can be:

          • Positive integer: 0-based index from the start (0, 1, 2, ...)
          • Negative integer: index from the end (-1 is last element, -2 is second-to-last, etc.)
          • Out of bounds: any index beyond array bounds returns Optional.None

        Returns Optional<Ar[number]>

        An Optional containing:

        • Optional.Some with the element if the index is valid
        • Optional.None if the index is out of bounds (including empty arrays)

        Example

        const users = [{ id: 1 }, { id: 2 }];
        const empty: { id: number }[] = [];

        const first = Arr.head(users);
        const none = Arr.head(empty);

        assert.deepStrictEqual(first, Optional.some({ id: 1 }));
        assert.deepStrictEqual(none, Optional.none);
